# 분석 보고서: 견적서 목록 및 조회 기능 (Quotation List & View)

## 1. 개요
현재 시스템은 견적서의 생성 및 저장 기능은 구현되어 있으나, 저장된 견적서들을 한눈에 확인하고 관리할 수 있는 '목록(List/Dashboard)' 페이지가 부재합니다. 사용자의 "발행한 견적서 정보를 확인할 수 있는 페이지" 요청에 따라, 대시보드 페이지 신설 및 조회 모드 최적화를 제안합니다.

## 2. 현재 상태 분석
- **루트 경로(`/`)**: 현재 신규 견적서 작성을 위한 편집기가 배치되어 있음.
- **상세 경로(`/quotation/[id]`)**: 저장된 데이터를 불러와 편집기에서 보여주지만, 발행된(`SENT`, `ACCEPTED`) 상태에서도 편집기 UI(사이드바 등)가 그대로 노출됨.
- **데이터 접근**: 목록을 가져오는 Server Action이 부재하며, 사용자가 본인의 견적서 목록을 확인할 방법이 없음.
- **요구사항 준수**: `spec.md`의 FR-006(상태에 따른 수정 제한)은 로직상 구현되어 있으나 UI적으로 '조회 모드'와 '편집 모드'의 구분이 모호함.

## 3. 문제점 및 개선 방향
### 3.1. 대시보드(목록) 페이지 부재
- **문제**: 사용자가 이전에 만든 견적서를 찾으려면 URL을 직접 입력해야 함.
- **해결**: `/` 경로를 견적서 목록 페이지로 전환. 제목, 상태, 총액, 생성일 등을 포함한 테이블/카드 형태의 UI 제공.

### 3.2. 일관되지 않은 사용자 경험
- **문제**: 신규 작성과 목록 확인의 구분이 없음.
- **해결**: '신규 견적서 작성' 버튼을 대시보드 상단에 배치하고, `/quotation/new` 또는 상태 초기화된 편집기 페이지로 유도.

### 3.3. 조회 모드 UI 미흡
- **문제**: 발행 완료된 견적서 조회 시에도 '블록 추가' 사이드바가 노출되어 문서의 전문성이 떨어짐.
- **해결**: `status !== 'DRAFT'`인 경우 사이드바를 제거하고, 중앙 집중형 레이아웃으로 변경.

## 4. 기술적 고려 사항
- **Server Actions**: `getQuotations` 함수를 추가하여 Prisma를 통해 `deletedAt`이 없는 견적서 목록을 조회.
- **Shared Components**: 대시보드에서 사용될 `StatusBadge`, `QuotationCard` 등 UI 컴포넌트 추출.
- **Layout**: Next.js의 App Router 특성을 활용하여 대시보드와 편집기 간의 레이아웃 전이 최적화.

## 5. 결론
본 분석을 바탕으로 견적서 목록 페이지를 신설하고, 기존 상세 페이지의 조회 모드를 강화하는 구현 계획을 수립할 것을 제안합니다.
