# 260313_quote-builder 프로젝트 분석 보고서

## 1. 프로젝트 개요
본 프로젝트는 사용자가 자유롭게 텍스트와 가격표 블록을 조합하여 전문적인 견적서를 작성하고, 이를 PDF로 내보내거나 DB에 저장할 수 있는 'AI 기반 견적서 빌더'입니다.

## 2. 현재 구현 상태 (As-Is)
- **UI/UX**: Toss 스타일의 미니멀한 디자인 시스템 적용. 반응형 레이아웃 및 드롭다운/입력 UI 완료.
- **Editor**: BlockNote 프레임워크 기반. 커스텀 블록(pricingTable)을 통해 표 형식의 가격 산정 기능 제공.
- **State Management**: Zustand를 사용하여 견적서의 제목과 블록 데이터를 중앙에서 관리.
- **Export**: html2canvas + jsPDF를 이용한 클라이언트 사이드 PDF 생성 로직 존재.
- **Database**: PostgreSQL 기반의 Prisma 스키마 정의 완료 (Quotation 모델).

## 3. 핵심 문제점 및 개선 필요 사항 (Pain Points)
### 3.1. 서버 사이드 로직 부재
- **API 연동 미비**: 견적서를 저장하거나 불러오는 실제 API 엔드포인트가 없음.
- **Prisma 활용도**: 모델은 정의되어 있으나 실제 서비스 로직에서 활용되지 않음.

### 3.2. 편집기 기능 고도화
- **블록 삽입 로직**: 외부 사이드바에서 블록을 추가하는 CustomEvent 기반 로직의 연동 확인 필요.
- **다양한 블록 부족**: 현재 텍스트와 가격표 위주이며, '서명 섹션', '회사 로고', '약관' 등 견적서 필수 요소 블록이 부족함.

### 3.3. PDF 품질 및 레이아웃
- **해상도 이슈**: html2canvas 캡처 방식의 한계로 인해 텍스트 선택이 불가능하며 인쇄 품질이 낮을 수 있음.
- **다단 레이아웃**: A4 규격에 최적화된 고정 레이아웃 가이드 부재.

## 4. 추가 제안 기능 (Proposed Features)
1. **AI 자동 완성**: 견적서 제목이나 품목명을 입력하면 AI가 적절한 가격대나 설명을 추천하는 기능.
2. **견적서 상태 관리**: '발행됨', '검토 중', '승인됨' 등 워크플로우 추적 기능.
3. **고객 전용 뷰어**: 생성된 견적서를 링크로 공유하고 고객이 온라인에서 바로 승인/서명할 수 있는 웹 뷰어.
4. **다국어/통화 지원**: 글로벌 비즈니스를 위한 통화 단위 변경 및 다국어 템플릿.

## 5. 결론 및 향후 계획
현재 프로젝트는 프론트엔드 UI와 편집기 핵심 로직이 매우 견고하게 구축되어 있습니다. 다음 단계로 서버 액션을 통한 데이터 영속성 확보와 전문적인 견적서 구성을 위한 블록 라이브러리 확장 혹은 AI 연동을 진행할 예정입니다.
