# Feature Specification: 견적서 생성 및 저장 기능 (Create and save quotation)

**Created**: 2026-03-17  
**Status**: Draft

## User Scenarios & Testing _(mandatory)_

### User Story 1 - 신규 견적서 작성 및 저장 (Priority: P1)

사용자는 새로운 프로젝트를 위한 견적서를 작성하고 시스템에 안전하게 저장할 수 있어야 합니다.

**Why this priority**: 가장 핵심적인 가치 제안이며, 다른 모든 기능(PDF 내보내기, 상태 관리 등)의 토대입니다.

**Independent Test**: 사용자가 견적서 제목을 입력하고 블록을 추가한 뒤 '저장' 버튼을 눌렀을 때, 데이터베이스에 정상적으로 반영되고 목록에서 확인할 수 있어야 합니다.

**Acceptance Scenarios**:

1. **Given** 빈 견적서 작성 화면에서, **When** 제목을 입력하고 텍스트 블록과 가격표 블록을 추가한 뒤 '저장' 버튼을 누르면, **Then** "저장되었습니다" 메시지가 표시되고 견적서 ID가 생성됩니다.
2. **Given** 저장된 견적서가 있는 상태에서, **When** 제목이나 블록 내용을 수정한 뒤 다시 '저장' 버튼을 누르면, **Then** 기존 데이터가 최신 내용으로 업데이트됩니다.

---

### User Story 2 - 가격표 블록을 통한 자동 금액 산출 (Priority: P2)

사용자는 견적서 내부에 가격표 블록을 추가하고, 품목별 단가와 수량을 입력하여 총액이 자동으로 계산되는 것을 볼 수 있어야 합니다.

**Why this priority**: 수동 계산의 번거로움과 오류를 줄여주는 핵심적인 편의 기능입니다.

**Independent Test**: 가격표 블록에 품목을 추가하고 수량을 변경했을 때, 하단의 합계 금액이 실시간으로 업데이트되는지 확인합니다.

**Acceptance Scenarios**:

1. **Given** 가격표 블록이 포함된 견적서에서, **When** 특정 항목의 수량을 '2'에서 '5'로 변경하면, **Then** 해당 항목의 소계와 견적서 전체 총액이 즉시 재계산됩니다.

---

### Edge Cases

- **빈 제목 저장**: 제목을 입력하지 않고 저장을 시도할 경우 사용자에게 알림을 주어야 합니다.
- **네트워크 오류**: 저장 도중 서버와의 연결이 끊겼을 때 데이터 손실 없이 사용자에게 재시도를 안내해야 합니다.
- **초대형 데이터**: 수백 개의 블록이 포함된 대규모 견적서도 성능 저하 없이 저장 및 로드되어야 합니다.

## Requirements _(mandatory)_

### Functional Requirements

- **FR-001**: 시스템은 견적서의 고유 ID, 제목, 블록 데이터, 총액, 상태 정보를 포함한 견적서 엔티티를 관리해야 합니다.
- **FR-002**: 사용자는 견적서 제목을 필수적으로 입력해야 하며, 텍스트(제목, 본문, 목록 등) 및 가격표 블록을 자유롭게 구성할 수 있어야 합니다.
- **FR-003**: 가격표 블록 내의 모든 금액(소계, 합계)은 단가와 수량을 기반으로 자동 계산되어야 합니다.
- **FR-004**: 견적서 저장 시 데이터베이스의 데이터 무결성을 위해 서버 측에서 총액(totalAmount)을 재검증해야 합니다.
- **FR-005**: 모든 견적서 데이터는 JSONB 형식으로 영구 저장되어야 합니다.

### Key Entities

- **Quotation**: 견적서의 기본 정보를 담는 엔티티 (ID, Title, Blocks, TotalAmount, Status, Timestamps).
- **Block**: 견적서를 구성하는 개별 콘텐츠 단위. 텍스트 유형과 가격표 유형이 존재합니다.

## Success Criteria _(mandatory)_

### Measurable Outcomes

- **SC-001**: 사용자는 견적서 작성 시작부터 저장 완료까지 30초 이내에 수행할 수 있어야 합니다. (초기 입력 기준)
- **SC-002**: 가격표 블록 내 수치 변경 시 합계 반영까지의 지연 시간은 200ms 미만이어야 합니다.
- **SC-003**: 모든 견적서 데이터는 저장 후 페이지를 새로고침했을 때 100% 일관되게 로드되어야 합니다.
- **SC-004**: 서버 측 총액 검증 통과율은 정상적인 요청 시 100%여야 합니다.

## Assumptions

- 사용자는 로그인이 된 상태이며, 본인의 견적서에 대해서만 읽기/쓰기 권한을 가집니다. (RLS 정책 적용 전제)
- 견적서 상태의 기본값은 'DRAFT'로 설정됩니다.
